# 数据治理与质量控制

为确保知识库可靠、可持续，建议遵循以下治理策略：

## 元数据标准

- **必填字段**：标题、作者、年份、来源、资源类型、教育阶段、主题标签。
- **可选字段**：DOI、URL、摘要、方法分类（如 GNN、Transformer）。
- 使用 [Dublin Core](https://www.dublincore.org/specifications/dublin-core/dcmi-terms/) 与 [EDAM Ontology](https://edamontology.org/) 的术语，保持跨平台兼容。

## 质量流程

1. **采集审查**：
   - 由领域专家审核科研类资源。
   - 教育资源需通过教师/教研员评估。
2. **版本管理**：
   - 使用 Git 追踪 YAML/JSON 数据文件的变化。
   - 重要更新标记 `version` 与 `last_reviewed` 字段。
3. **自动化校验**：
   - `ingestion/loader.py` 内置模式验证与重复检测。
   - 构建 CI 流水线运行单元测试与数据完整性检查。

## 合规性

- 遵守出版商版权政策，优先链接原始页面而非直接分发全文。
- 对于学生数据或作业成果，需进行匿名化处理。
- 在 API 返回中明确引用来源，避免信息误导。

## 安全与隐私

- 生产环境数据库建议部署在 VPC 内部或使用托管数据库服务。
- 对 API 设置访问令牌和速率限制，防止滥用。
- 定期备份数据库与索引文件，确保可恢复性。
